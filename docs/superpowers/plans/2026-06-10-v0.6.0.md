### Task 2: tree merge + changed tracking

**Files:** `internal/ui/tree.go`, `tree_test.go`.

- `node` gains `changedAt time.Time`.
- `func (t *tree) mergeChildren(parent *node, entries []fs.Entry, now time.Time)` — parent nil = merge roots. Build the new child list from `entries` (already sorted by fs.Sort): for each entry, if a node of the same Name exists in the old list, KEEP that node (preserving expanded/loaded/children/changedAt and its own descendants), update its entry, and set `changedAt = now` when Size or ModTime differ; if no old node, create one with `changedAt = now`. Old nodes not in entries are dropped. Depths/parent pointers maintained. Reflatten at the end (cursor clamps).
  - Important nuance: the very first merge after a fresh load must NOT mark everything changed — merging identical entries (same size/mtime) over existing nodes marks nothing; brand-new nodes are only "changes" when the tree already had that directory loaded. To keep it simple and correct: `mergeChildren` only ever runs on directories that were already loaded — new nodes there ARE genuinely new files. (setRoot/setChildren remain the initial-load path and never mark.)
- `func (t *tree) expandedDirs() []*node` — root-implied plus every loaded+expanded dir node currently visible-or-not (spec says root + expanded; collapsed-but-expanded-underneath is fine to include or skip — pick: walk all loaded nodes with expanded=true regardless of visibility, simplest recursive collection).
- `func (t *tree) recentChanges(within time.Duration, now time.Time) []*node` — all nodes (recursive) with `now.Sub(changedAt) < within`, most recent first.
- Tests: merge preserves expansion + cursor; mtime delta marks; size delta marks; new entry marked + inserted in order; vanished dropped + cursor clamp; recentChanges ordering; expandedDirs collection.

Commit: `feat: tree merge with change tracking`

### Task 3: watch mode in the explorer

**Files:** `internal/ui/explorer.go`, `messages.go`, `app.go` (tick scheduling goes through explorer's returned cmds; App routes the new msgs like the others — check: non-key msgs already route to explorer), `explorer_test.go`, `styles.go` (changed-highlight style, e.g. foreground 214 orange).

- Constants: `watchInterval = 3 * time.Second`, `changedHighlight = 45 * time.Second`, in explorer.go.
- New msgs/cmds in messages.go:

```go
type watchTickMsg struct{}

type refreshedMsg struct {
	parent  *node // nil = root
	path    string
	entries []fs.Entry
	err     error
}

func watchTickCmd() tea.Cmd {
	return tea.Tick(watchInterval, func(time.Time) tea.Msg { return watchTickMsg{} })
}

func refreshCmd(fsys fs.Filesystem, parent *node, path string) tea.Cmd {
	return func() tea.Msg {
		entries, err := fsys.List(path)
		return refreshedMsg{parent: parent, path: path, entries: entries, err: err}
	}
}
```

- explorer gains `watch bool` (default true, set in newExplorer). First tick scheduled from Init (`tea.Batch(loadRootCmd(...), watchTickCmd())`).
- On `watchTickMsg`: if !watch → return nil cmd (tick chain stops; toggling back on restarts it). If watch: build batch of `refreshCmd` for root (parent nil, e.rootPath) + every `e.tree.expandedDirs()` node (path = node entry path), PLUS `watchTickCmd()` to schedule the next tick, PLUS the git refresh cmd from Task 4 (left as a seam: a method `e.gitRefreshCmd() tea.Cmd` returning nil until Task 4).
- On `refreshedMsg`: err → `e.watch = false` + red status "watch off: <err>" (only once — subsequent refreshedMsg errors from the same batch: acceptable to set the same status). Success → `e.tree.mergeChildren(msg.parent, msg.entries, time.Now())`. Guard staleness: if `msg.parent != nil && !msg.parent.loaded` ignore; also ignore when msg.path no longer matches (root refresh racing a path jump: `msg.parent == nil && msg.path != e.rootPath` → drop).
- `w` key in modeTree: toggles watch; turning ON also returns `watchTickCmd()`; status "watch on"/"watch off".
- Rendering in renderNode: `if !n.changedAt.IsZero() && time.Since(n.changedAt) < changedHighlight` → changed style + `"● "` marker replacing the two-space file marker (dirs keep ▸/▾, the name gets the changed color). Changed style wins over git/dir/dim colors (cursor still wins over everything).
- setFilesystem / path jumps keep watch state as-is.
- Tests (Update-level, drive msgs by hand — do NOT wait real seconds; never call watchTickCmd()'s returned cmd in tests, construct watchTickMsg{} directly): tick with watch on returns a batch containing a refresh for root and for an expanded dir and not for collapsed ones; tick with watch off returns nil; refreshedMsg with changed mtime marks the node (assert via tree state); refreshedMsg error sets watch=false + error status; `w` toggle both ways; stale refreshedMsg (root path mismatch) ignored.

Commit: `feat: watch mode with changed-file highlights`
