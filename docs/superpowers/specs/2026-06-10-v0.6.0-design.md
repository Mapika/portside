# portside v0.6.0 — "the agent cockpit" — Design Spec

**Date:** 2026-06-10
**Status:** Approved

## Purpose

Make the explorer aware of *what is happening* to the files while an AI agent
(or anyone) works on them:

1. **Watch mode** — the tree refreshes itself and highlights recently changed
   files, so you can see what the agent in the next pane is touching.
2. **Git status colors** — modified/untracked coloring in the tree, local and
   remote.
3. **Send recent changes to the agent** — `C` types the list of recently
   changed paths into the agent pane.

## 1. Watch mode

- `fs.Entry` gains `Size int64` and `ModTime time.Time`; both backends fill
  them in `List` (local: `DirEntry.Info()`, sftp: already has `FileInfo`).
- A `tea.Tick` every 3s (while watch is on) re-lists the root and every
  expanded directory, then **merges** into the existing tree:
  - existing nodes keep their state (expanded/loaded/children) — matched by
    name within a directory;
  - an entry that is new, or whose Size/ModTime changed, gets
    `changedAt = now`;
  - vanished entries are dropped (cursor clamps as usual).
- Rendering: a node with `time.Since(changedAt) < 45s` is highlighted
  (distinct color + `●` marker in front of the name).
- `w` toggles watch; default **on**. Status confirms "watch on/off".
- A refresh that errors (connection died, dir gone) turns watch off with one
  red status message — no error spam every tick.
- Refresh only re-lists visible (root + expanded) directories; one async
  batch per tick.

## 2. Git status colors

- Optional capability interface in fs:

```go
// Execer is implemented by backends that can run a command on the machine
// they browse.
type Execer interface {
	Exec(name string, args ...string) ([]byte, error)
}
```

  - `Local.Exec` → `exec.Command(name, args...)` (argv form, no shell).
  - `SFTP` gains an optional `*ssh.Client` (new constructor
    `NewSFTPWithExec(sftpClient, sshClient, host)` — plain `NewSFTP` keeps
    working without exec). Remote `Exec` runs one ssh session with the argv
    list joined with proper single-quote shell escaping per argument.
- On each watch tick (and on root load), if the backend implements Execer:
  - resolve the repo root once per rootPath:
    `git -C <root> rev-parse --show-toplevel` (cached; a failure caches
    "not a repo" — re-checked only when rootPath changes);
  - `git -C <root> status --porcelain -z`, parsed into
    `map[absPath]gitState` where state is `gitUntracked` ("??"/"!!"→skip
    ignored) or `gitModified` (anything else).
- Rendering: untracked → green name, modified → yellow name. Changed-recently
  highlight (watch) wins over git color. Directories are not colored in v1.
- No git on the machine / not a repo / exec errors → silently no colors.

## 3. Send recent changes (`C`)

- `C` collects paths of nodes currently marked changed (watch highlights,
  most recent first, cap 20), joins with spaces, and sends through the same
  channel as `c` (tmux pane / clipboard; same control-char guard per path —
  offending paths are skipped).
- Nothing changed → status "no recent changes".

## Out of scope

inotify/fsnotify (polling only), branch/ahead-behind info, git colors for
directories, diff viewing, configurable tick interval or highlight duration
(constants: 3s / 45s), watching collapsed subtrees.

## Testing

- fs: Entry ModTime/Size populated (both backends); Local.Exec runs argv;
  SFTP Exec via in-process server — NOTE: the gliderlabs test server's
  session handler currently ignores exec requests; extend it to execute the
  command locally and return output (guard: test-only package).
- tree: merge keeps expansion/cursor, marks changed on mtime/size delta,
  marks new entries, drops vanished ones.
- ui: tick → refresh cmds for expanded dirs only; `w` toggle; refresh error
  turns watch off; git map applied to rendering (assert via a render or
  state, whichever is testable); `C` with and without recent changes.
- git parsing: porcelain -z fixture bytes → map (pure function test).
